Git 操作心得

  • 日常开发分支切换提交
  1. 先从最新的 master 分支切一个开发分支 develop_master 出来
  2. 将新切出来的分支推送到远程 origin
  3. 从新分支切一个自己的本地分支 develop_local
  4. 在本地的分支上进行开发
  5. 如果要提交，先更新远程分支 develop_master
  6. 在自己的本地分支 develop_local 进行 git rebase develop_master 变基操作
  7. 切换到远程分支 develop_master 将本地分支合并到远程分支上 git merge --no-ff develop_local 并推送 git push
  8. 删除自己的本地分支 git branch -D develop_local

之后每次修改分支上的内容最好从最新的远程分支切换新分支，这样可以保证我们每次的变基操作不会影响其他人的提交
