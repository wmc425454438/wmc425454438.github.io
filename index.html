Git 操作心得

Git 操作心得

  • 日常开发分支切换提交
  1. 先从最新的 master 分支切一个开发分支 develop_master 出来
  2. 将新切出来的分支推送到远程 origin
  3. 从新分支切一个自己的本地分支 develop_local
  4. 在本地的分支上进行开发
  5. 如果要提交，先更新远程分支 develop_master
  6. 在自己的本地分支 develop_local 进行 git rebase develop_master 变基操作
  7. 切换到远程分支 develop_master 将本地分支合并到远程分支上 git merge --no-ff develop_local 并推送 git push
  8. 删除自己的本地分支 git branch -D develop_local

之后每次修改分支上的内容最好从最新的远程分支切换新分支，这样可以保证我们每次的变基操作不会影响其他人的提交

注意官方提示的rebase的操作提示，再变基前要确认没有将该分支推送或者合并到远端

例如我用本地分支变基过master，提交了新的commit到远端并合并到了测试分支uat。之后本地分支不能再变基master了。只能将master合并到本地分支。并且，本地切换出来的开发分支必须在变基本地分支之后及时删除。保证每次提交后用的都是最新分支。

变基的好处就是可以将自己的commit可以放到最前面。清晰的看出来当前commit的操作。

  • Rebase -i 合并commit

一般用于本地开发时，还未推送到远端时操作，可以将几个commit合并成一个。用于操作git的操作记录是一个比较好的手段。

打家劫舍

问题描述

小偷进行偷窃，不能偷窃相邻的两个房间，不然会触发报警，请问怎样才能窃取到最大的金额。

金额数字用数组表示

动态规划的又一道基础题，先分析问题子结构，再列公式。

  • 要首先看一个问题能不能由他的子问题进行解决

    • 可以：能用动态规划；不可以：不能用动态规划解决。

  • 假设一共有n个房间，求偷n个房间的最大值

  • 最后一个房间只有偷和不偷两种方式

    • 偷：f(n) = f(n-2) + n
    • 不偷: f(n) = f(n-1)

  • 所以，n个房间的最大值

    • f(n) = max(f(n-2) + n, f(n-1))

  • 基本思想如上所述，该问题可以由动态规划算法完成。

仔细观察可以发现，算到n个房间只需要n-1和n-2的最大值就可以了，也就是我们只需要保存前面两个情况的最大值。

就可以算出n个房间的最大值，所以我们只需要利用变量重复赋值就可以了，不需要用到数组。

这样我们的空间复杂度就由O(N)变成了O(1)。

宏观和微观任务

概览

宏观任务和微观任务（Macro Task and Micro Task）

Js 中，有两类任务队列：宏任务队列（macro tasks）和微任务队列（micro tasks）。宏任务队列可以有多个，微任务队列只有一个。那么什么任务，会分到哪个队列呢？

  • 宏任务：script（全局任务）, setTimeout, setInterval, setImmediate, I/O, UI rendering.
  • 微任务：process.nextTick, Promise, Object.observer, MutationObserver.
  • 微任务始终先于宏任务执行。

这里我们强制了 1 秒的执行耗时，这样，我们可以确保任务 c2 是在d之后添加到任务队列。

输出顺序abc

分析异步执行的顺序

  • 首先分析有多少个宏任务
  • 在每个宏任务中，分析有多少个微任务
  • 根据调用次序，确定宏任务中的微任务和执行次序
  • 根据宏任务的触发规则和调用次序，确定宏任务的执行次序
  • 确定整个顺序

上面之所以加50ms的阻塞，是因为 setTimeout 的 delayTime 最少是 4ms. 为了避免认为 setTimeout 是因为4ms的延迟而后面才被执行的，我们加了50ms阻塞。
