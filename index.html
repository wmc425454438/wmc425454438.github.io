动态规划

基本概念

动态规划英文 Dynamic Programming，是求解决策过程最优化的数学方法，后来沿用到了编程领域。
动态规划的大致思路是把一个复杂的问题转化成一个分阶段逐步递推的过程，从简单的初始状态一步一步递推，最终得到复杂问题的最优解。
动态规划解决问题的过程分为两步：

  1. 寻找状态转移方程
  2. 利用状态转移方程式自底向上求解问题

爬楼梯问题

挖金矿问题

假设有n个金矿，w个工人，每座金矿需要的工人和金矿量分别用p[]和g[]来表示。
问：所有工人挖去金矿的最大值。

由于最后一座金矿有两种情况：挖或者不挖。

  • 最大子结构有两种情况：

    • w个工人挖n-1个金矿
    • w-p[n-1]个工人挖n-1个金矿 + g[n-1]

  • 边界情况

    • 人手不够，不够挖一座金矿，或者没有金矿
    • 所有人挖第一座金矿

状态转移方程

  • f(n, w) = 0 (n <= 1, w < p[0]) 无金矿或者人手不足
  • f(n, w) = g[0] (n == 1, w >= p[0]) 人手足够，只挖第一个金矿
  • f(n, w) = f(n-1, w) (n > 1, w < p[n-1]) 剩余人数少于这个金矿的所需人数
  • f(n, w) = max(f(n-1, w), f(n-1, w - p[n-1]) + g[n-1]) (n > 1, w >= p[n-1]) 工人充足的情况

工人数较大时，需要开辟的空间单位会变大，这时使用递归反而会变得简单

Javascript Template Mode

テンプレートメソッドパターンは、パッケージの変更によりシステムのスケーラビリティを向上させる典型的な設計パターンです。オブジェクト指向言語では、テンプレートメソッドパターンを使用するプログラムでは、サブクラスのメソッドタイプと実行順序は変わらないため、ロジックのこの部分は、親クラスのテンプレートメソッドに抽象化されます。サブクラスメソッドの実装方法は変更可能なため、これを採用します部分的に変更されたロジックはサブクラスにカプセル化されます。新しいサブクラスを追加することにより、システムに新しい関数を変更せずに追加できます。抽象親クラスと他のサブクラスを移動することも、開放性と閉鎖性の原則に沿っています。
しかし、JavaScriptでは、テンプレートメソッドパターン、高階関数を実装するために同じパターンに従う必要はありません。より良い選択です。
